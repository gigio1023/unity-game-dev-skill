# Assets and Licensing

Use this reference before recommending, downloading, importing, or redistributing
third-party assets.

## Required checks

- Identify the exact asset, author, canonical source, and current download.
- Read the license attached to that version; do not infer it from a site or tag.
- Check commercial use, modification, redistribution, attribution, trademark,
  model-release, and AI-training restrictions relevant to the intended use.
- Preserve a license copy, source URL, author, version/date, and required credit
  in the project's chosen attribution record.
- Check package contents and import settings before adding them to the project.

“Free,” “royalty-free,” and “CC0” are not interchangeable. Do not call an asset
“risk-free.” State the verified permission and any remaining ambiguity.

Importing or downloading content is an external side effect and may add many
files or packages. Obtain authority before doing it. Prefer existing project
assets when they satisfy the need.

For placeholder work, distinguish assets intended only for local prototyping
from content safe to ship or redistribute.
